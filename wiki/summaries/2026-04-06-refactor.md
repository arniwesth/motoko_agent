# Refactor: Move Source Code into Unified src/ Directory

This session restructured the repository by moving `core/`, `tui/`, and `examples/` under a single `src/` directory. The goal was to consolidate the project's own source code into one location, improving discoverability and adherence to common monorepo conventions.

## Key Changes

- **Directory moves:**
  - `core/*.ail` → `src/core/*.ail` (module paths updated)
  - `tui/` → `src/tui/`
  - `examples/` → `src/examples/`

- **Updated files:**
  - `Makefile` – all targets (check_core, build_tui, test_core, run_hello_world) now point to new paths
  - Documentation: `AGENTS.md`, `SYSTEM.md`, `README.md`, `CLAUDE.md` – diagrams, setup steps, and workflow references realigned
  - Scripts: `scripts/run-agent.sh`, `scripts/install-prerequisites.sh` – entry point and dependency paths changed to `src/tui/`
  - Source files: `src/tui/src/runtime-process.ts`, `src/tui/src/index.ts` – internal references updated

## Verification

- `make check_core` passes all 11 tests
- `cd src/tui && npm run build` completes successfully

## Impact

This refactor establishes a clear [[concepts/src-directory-pattern]] that separates the project's own code from configuration, docs, and external dependencies. It simplifies the [[concepts/build-system]] targets by reducing path duplication and lays groundwork for future workspace tooling.
