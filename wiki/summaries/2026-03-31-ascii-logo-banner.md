# ASCII Logo Banner in TUI Header

## Goal

Integrate the ASCII-art logo (generated by `imageToAscii.ts` from `input.png`) into the agent's startup header in the TUI.

## Implementation

### Banner Generation
- A regeneration script, `gen-banner.mjs`, executes `imageToAscii.ts` directly and captures its stdout to guarantee byte-identical output.
- The script writes `tui/src/banner.ts`, exporting a `BANNER: string[]` array.
- `JSON.stringify` transforms raw ESC bytes (0x1b) into `\u001b` sequences, which TypeScript preserves at runtime (see [[concepts/ANSI Escape Sequence Handling]]).
- The banner uses dual-block characters (▀ and ▄) for full colour fidelity (see [[concepts/Dual-Block ASCII Rendering]]).

### Integration Points
- **Non-TTY mode** (`tui/src/index.ts`): The `BANNER` is printed unconditionally to stdout in `main()`, followed by the version string. This ensures the logo appears even when piping or using `PlainLogger`.
- **TTY mode** (`tui/src/ui.ts`): The `BANNER` is imported and rendered as `Text` children at the top of the history pane in the `AgentUI` constructor. The version markdown heading follows the logo.

## Pitfalls & Fixes
| Issue | Fix |
|-------|-----|
| Wrong logo (framework logo used) | Changed path to `input.png` |
| Missing ▄ character, circular artefacts | Captured `imageToAscii.ts` stdout directly instead of reimplementing |
| Broken ANSI escapes (`\\x1b` literal) | Used `JSON.stringify` to produce correct `\u001b` |
| Logo missing in non‑TTY mode after cleanup | Restored unconditional banner print in `main()` |

## Key Files
- `gen-banner.mjs` — regeneration script (see [[concepts/Banner Regeneration Script]])
- `tui/src/banner.ts` — auto-generated banner data
- `tui/src/index.ts` — main entry point, prints banner before logger branch
- `tui/src/ui.ts` — TUI history pane rendering
