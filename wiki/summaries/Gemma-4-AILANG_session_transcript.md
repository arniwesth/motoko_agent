# Summary: Gemma-4-AILANG Fine-Tuning Plan

This document captures a strategic deep-dive session that turns [[concepts/ailang]]'s "Lever 7 (Fine-Tuning / LoRA)" into a concrete plan for the Kaggle "Gemma 4 Good" hackathon. The core objective is to create an open-weights model that **natively writes mathematically verified AILANG code** by violently overwriting Gemma 4's Python/JS priors with pure functional logic and Z3-verified contracts.

## Key Decisions

- **Native raw-code author** (not tool-call sequences) – flashier for the hackathon, harder but more rewarding.  
- **Synthetic dataset** generated by a stronger frontier model (Claude Opus 4.6) via [[concepts/reasoning-distillation]].  
- **No Kaggle hardware constraint** – the project will use rented cloud GPUs (8x H100 eventually).  
- **Open-source only** – Unsloth Pro/Enterprise ruled out; training stack uses Axolotl, Llama-Factory, or Unsloth’s free tier.  
- **Benchmark-driven** – success measured against official AILANG benchmarks, especially the hard [[concepts/z3-contract-verification]] tasks.

## Project Phases

### Phase 0 – Micro-Pilot (<$25)
- 1× A100 (80GB) on RunPod/Lambda.  
- Unsloth free OSS + QLoRA (4-bit).  
- 500 synthetic examples generated by Claude Opus.  
- Success criterion: >40% parse rate and at least one compiling Z3 contract.

### Phase 1 – Synthetic Data Generation
- 15,000–20,000 reasoning-distilled trajectories via Claude Opus.  
- Composition: 40% Z3 contracts, 30% core syntax/effects, 30% error‑recovery (multi-turn corrections).  
- Output format uses Gemma 4’s `<|think|>` tags to bake rule-based reasoning into weights.

### Phase 2 – Full-Scale Training
- 8× H100 (80GB) bare-metal instance (~$24/hr).  
- Axolotl or Llama-Factory with PyTorch FSDP / DeepSpeed ZeRO-3.  
- Native `bfloat16`, Flash Attention 2, no quantization.  
- High-rank LoRA (`r=256–512`) targeting all linear layers, or Full Fine-Tuning.  
- RoPE scaling to 8192 tokens for long Z3 contracts.

### Phases 3–4 – Evaluation & Submission
- Custom harness runs `ailang check` and `ailang verify` on benchmark prompts.  
- Target: beat Claude Opus on contract benchmarks like `Contract Rle Roundtrip` and `Type Unify`.  
- Deliverables: Hugging Face model + LoRA adapter, open-sourced synthetic dataset, Kaggle inference notebook, and a write‑up framing the “distribution shift” problem.

## Data Generation Script
An included Python script (`generate_phase0_data.py`) orchestrates Claude Opus calls via Anthropic or OpenRouter. It auto-generates diverse AILANG prompts, parses `<thought_process>` and `<ailang_code>` XML blocks, and writes instruction-tuning JSONL with Gemma `<|think|>` formatting.
