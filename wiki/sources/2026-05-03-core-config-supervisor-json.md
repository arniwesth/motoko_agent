# Motoko Core Config Supervisor JSON Migration

Date: 2026-05-03

## Context

The session started by reading `README.md` and `v0.12.1.md` to refresh the
Motoko architecture and current AILANG syntax/stdlib constraints. The requested
work was to implement `.agent/plans/Motoko_Core_Config_Supervisor.md`, whose
original Phase 1 assumed adding TOML parsing to AILANG via `std/toml` or a
Go-backed `_toml_parse` builtin.

## Plan Change

An initial attempt began to implement `std/toml` in the vendored `ailang/`
runtime. The user interrupted and clarified that touching `ailang/` is out of
scope for this task. Those runtime changes were backed out immediately, and a
diff check confirmed no remaining changes under `ailang/`.

We then discussed alternatives. The key decision was:

- Do not modify `ailang/`.
- Do not hand-roll a TOML parser in AILANG.
- Move Motoko core-owned profile config to JSON, because AILANG already has
  `std/json`.
- Keep TOML as legacy TypeScript TUI compatibility during migration.

The plan file was updated accordingly:

- Added explicit non-goal forbidding `ailang/` changes for this task.
- Replaced the TOML parser phase with a JSON profile config phase.
- Made `.motoko/config/<profile>/*.json` the headless/core source of truth.
- Required TOML-only profiles to emit actionable warnings rather than being
  parsed by core.
- Updated precedence, tests, backend examples, risks, and verification notes to
  refer to profile JSON.

## Implementation

Added `src/core/config.ail`:

- Defines typed config records: `AgentConfig`, `BackendConfig`, `ToolConfig`,
  `ExtensionConfig`, `ComposeConfig`, `ContextModeConfig`, `InvocationConfig`,
  and `RuntimeConfig`.
- Loads `.motoko/config/<profile>/*.json` using `std/json`.
- Supports default profile, named profile, absolute profile path, and flat
  `.motoko/config.json` fallback.
- Implements precedence:
  `hardcoded defaults < .env/.export < profile JSON < shell env`.
- Keeps invocation-specific values such as `TASK` outside profile JSON.
- Emits warnings for invalid JSON and legacy TOML-only config.
- Exposes `print_config_json`.

Added `src/core/backend.ail`:

- Defines `BackendHandle`.
- Adds `start_or_connect_backend`, `exec_backend`, and `stop_backend`.
- Supports `external_http` and `none` modes.
- Starts the TypeScript backend through a configured command/args path for the
  migration phase.

Added `src/core/supervisor.ail`:

- Loads runtime/invocation config.
- Emits config warnings.
- Starts or connects to the configured backend.
- Calls the real RPC agent loop via `run_with_config`.

Updated `src/core/rpc.ail`:

- Added exported `run_with_config(cfg, inv)`.
- `main()` now loads config and delegates to `run_with_config`.
- Startup now uses typed config for backend URL, model, workdir, hybrid tools,
  step delay, system prompt, extension order, and extension strictness.
- `session_start` now includes `config_profile`, `config_dir`, and
  `backend_mode`.

Updated `src/core/ext/runtime.ail`:

- Added `init_runtime_with_config(order, strict)` so extension order/strictness
  can come from typed config instead of direct env reads.

Added TypeScript/backend migration pieces:

- `src/tui/src/env-server-main.ts`, a standalone backend entrypoint that starts
  the existing `env-server.ts` without importing or starting the TUI.
- JSON config templates in `src/tui/src/config.ts`.
- `src/tui/src/init-config.ts` now writes JSON config and legacy TOML config.
- Built `src/tui/dist/*` output was regenerated by `bun run build`.

Added default JSON profile files:

- `.motoko/config/default/config.json`
- `.motoko/config/default/compose.json`
- `.motoko/config/default/context_mode.json`
- `.motoko/config/default/exa_search.json`
- `.motoko/config/default/omnigraph.json`

Updated `README.md`:

- Documents JSON as the core-owned config format.
- Notes TOML as legacy TUI compatibility during migration.
- Updates examples and precedence to use `.json`.

## Verification

Passed:

```bash
ailang check src/core/config.ail
ailang check src/core/backend.ail
ailang check src/core/supervisor.ail
ailang check src/core/rpc.ail
ailang run --entry print_config_json --caps IO,FS,Env src/core/config.ail
cd src/tui && bun run build
cd src/tui && bun test src/config.test.ts
ailang test src/core/parse_test.ail
ailang test src/core/agents_md.ail
```

Observed unrelated behavior:

```bash
ailang test src/core/parse.ail
```

reported `No tests found` and exited nonzero. This appears unrelated to the
config/supervisor changes.

Confirmed:

- No remaining diff under `ailang/`.
- The config JSON entrypoint emits normalized config from
  `.motoko/config/default/config.json`.
- The supervisor emits backend startup and then runs through the real RPC path
  rather than remaining a metadata-only stub.

## Current State

The implementation is an incremental migration slice, not the final end state.
The TypeScript backend remains the execution backend, now available through a
standalone entrypoint. The TUI still has legacy TOML compatibility. Core-owned
config now has a JSON path that AILANG can parse without runtime changes.

Untracked directories (`DR-Venus/`, `ailang/`, `little-coder/`,
`polyglot-benchmark/`) were pre-existing workspace state and were not modified
for this task.
