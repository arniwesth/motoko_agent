# ASCII Logo Banner in TUI Header

## Goal
Integrate the ASCII-art logo (generated by `imageToAscii.ts` from `input.png`) into the agent's TUI startup header.

## What Was Done

### 1. Banner Generation (`gen-banner.mjs`)
- Created `gen-banner.mjs` — a regeneration script that **runs `imageToAscii.ts` directly** and captures its stdout. No reimplemented logic — guarantees byte-identical output every time.
- The script writes `tui/src/banner.ts`, exporting a `BANNER: string[]` array.
- `JSON.stringify` converts real ESC bytes (0x1b) to `\u001b` sequences, which TypeScript preserves correctly at runtime.

### 2. `tui/src/banner.ts` (Auto-generated)
- Contains the ASCII-art logo from `input.png` at 30 columns wide, with full ANSI true-colour escape sequences.
- Includes both ▀ (UPPER HALF BLOCK) and ▄ (LOWER HALF BLOCK) characters — the dual-block pixel encoding that `imageToAscii.ts` uses.

### 3. `tui/src/index.ts` — Main Entry Point
- Imports `BANNER` and prints it to stdout unconditionally in `main()` before the TTY/PlainLogger branch split.
- This covers both paths: the `AgentUI` TUI renders the banner in its history pane for TTY mode, and the stdout print covers non-TTY mode (`PlainLogger` / piped).
- Version string ("AILANG Agent v0.1.0") follows the banner.

### 4. `tui/src/ui.ts` — TUI History Pane
- Imports `BANNER` and renders each line as a `Text` child at the top of the history pane in the `AgentUI` constructor.
- The version markdown heading ("## AILANG Agent v...") follows the logo in the history.

### Pitfalls & Fixes
| Issue | Cause | Fix |
|---|---|---|
| Wrong logo (framework logo instead of `input.png`) | Initial generator hardcoded `ailang/ui/public/logo.png` path | Changed to `input.png` (the agent's own logo) |
| Only ▀ character, no ▄ — circular artefacts | First `gen-banner.mjs` only used UPPER half-block | Rewrote to capture `imageToAscii.ts` stdout directly |
| Broken ANSI escapes (`[48;` instead of `\x1b[48;`) | `tsStr()` emitted `\\x1b` (literal text) instead of `\u001b` | Using `JSON.stringify` which correctly maps 0x1b → `\u001b` |
| `make run` stopped showing the logo | Cleanup removed `BANNER` print from `index.ts`, but non-TTY mode never reaches `AgentUI` | Restored unconditional `BANNER` print in `main()` |
| Dead import warnings | `BANNER` import in `index.ts` was removed then needed again | Kept the import; it is used by `main()` for the stdout banner print |
